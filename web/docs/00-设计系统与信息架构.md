# postern 控制台 · 设计系统与信息架构

> 本文是 postern 控制面管理前端（SPA）的**统一设计基座**：技术栈、设计系统（视觉令牌 + 组件 + 交互模式）、信息架构与页面清单。所有页面详细设计（`01`~`NN`）在此基座之上展开，必须遵守同一套令牌、组件与交互约定。**纯设计，不含实现代码。**

---

## 〇、定位与受众

- **是什么**：postern 控制面（control plane）的**人类运维管理控制台**——管理策略状态（主体/角色/资源/授权/凭证/细则/条件/模式/拒绝指引/审批/设置），查询审计、分析拒绝、触发红队自检。
- **受众**：运维/安全管理员（**人**，非 Agent）。Agent 物理不可达控制面（`control.sock=0600`）。
- **核心心智**：**预设决定一切，拒绝引导一切，人只修订预设。** 控制台不是"实时审批台"，而是"**规则编辑器 + 观测台 + 应急闸**"。绝大多数操作是**修订规则**（持久、异步），而非批准单次请求。

## 一、技术栈（已定，理由见下）

| 层 | 选型 | 理由 |
|---|---|---|
| 框架 | **React 18 + TypeScript（strict）** | 生态成熟、类型安全、团队通用 |
| 构建 | **Vite** | 快、零配置、HMR |
| 样式 | **Tailwind CSS** + CSS 变量令牌 | 令牌驱动、约束一致性、无样式漂移 |
| 路由 | **React Router v6** | 标准 SPA 路由 |
| 数据 | **TanStack Query（React Query）** | 控制面 API 的缓存/失效/乐观更新/分页/重试统一收口 |
| 表单 | **React Hook Form + Zod** | 类型化校验、与后端契约对齐 |
| 图标 | **lucide-react** | 线性、克制、与安全工具气质一致 |
| 测试 | **Vitest + Testing Library**（单元/组件）、**Playwright**（统一 e2e） | 与统一 e2e 要求对齐 |
| 包管理 | **pnpm** | 快、磁盘友好 |

**目录**：`web/`（SPA 根）。`web/src/{app,pages,components,api,lib,styles}`、`web/docs/`（本设计集）、`web/e2e/`（Playwright）。

**控制面接入（实现期决策，不阻塞本设计）**：浏览器无法直连 UDS。开发期 Vite dev-server 经一个**本地桥**把 `/v1/*` 代理到 `control.sock`；生产期由桌面壳（或 daemon 侧本地静态服务 + 桥）承载。SPA 只面向 `/v1/*` HTTP/JSON 契约编程，与传输无关。

## 二、设计原则（七条，贯穿所有页面）

1. **fail-closed 也体现在 UI**：任何不确定状态（加载中、出错、权限不足、数据缺失）默认呈现为**受限/拒绝态**，绝不乐观假装成功。写操作失败→明确红色错误 + 不改本地视图。
2. **真话且只说事实**：UI 只呈现策略与系统的客观事实 + 资源所有者预写内容（deny_notes 原样转述）；**绝不替系统生成"建议话术"**。denials 分析只呈现聚合模式（"某主体反复被拒 X"），调整方向由人裁决。
3. **规则编辑器气质**：以**修订规则**为中心（持久、可审、带 diff/确认），而非"批准请求"。破坏性/扩权操作（授权、freeze、shutdown、import 覆盖）一律二次确认 + 摘要预览。
4. **应急可达**：**freeze（一键拉闸）**与**吊销凭证**必须全局随手可达（顶栏常驻），≤2 步触达，醒目但防误触（确认）。
5. **密度优先、信息层级清晰**：管理控制台是数据密集型——紧凑表格、可排序/筛选/分页（强制分页，对齐 `DB_PAGINATION_MANDATORY`）、关键状态用色/徽章一眼可辨。
6. **匿名化/脱敏在前端也成立**：UI 永不显示真实地址/凭据明文；凭证只显示元数据（kind/可信域/有效期/吊销态/`secret_hash` 不显示）；资源只显示代号。
7. **可追溯**：每个写操作成功后提示其落库后果（policy_rev 前进、三联动）、可跳转到对应 audit 事件；乐观锁冲突（409）明确提示"他人已改、请刷新重试"。

## 三、视觉设计系统（令牌）

**气质**：安全运维控制台——克制、专业、暗色优先、低饱和中性灰 + 语义强调色。**不花哨**（公理六的视觉对应：不渲染、不卖弄）。

### 3.1 色彩令牌（语义优先，暗/亮双主题，CSS 变量）
- **中性**：`--bg`（底）/`--surface`（卡片/表格）/`--surface-2`（嵌套）/`--border`/`--text`/`--text-muted`。暗色为默认（运维夜间友好），亮色为可选。
- **语义状态**（贯穿全站，含义固定不可复用作装饰）：
  - `--allow`（绿）：allow / 健康 / PASS / 生效。
  - `--deny`（红）：deny / 错误 / FAIL / 吊销 / 危险动词（destroy）。
  - `--warn`（琥珀）：escalate / 过期临近 / 降级 / 维护模式 / 需注意。
  - `--info`（蓝）：信息 / 链接 / observe。
  - `--freeze`（红=强调，带斜纹/脉冲微动）：**冻结态全局横幅**专用，最高视觉优先级。
- **能力动词配色**（一致映射，徽章用）：observe=蓝灰、query=蓝、mutate=琥珀、execute=橙、manage=紫、destroy=红。**危险度递增=色温递增**。
- **模式配色**：normal=中性、observe=蓝、maintain=琥珀、freeze=红。

### 3.2 排版
- 正文 `Inter`（或系统无衬线）；**等宽 `JetBrains Mono`** 用于：雪花 id、hash 摘要、JSONL、SQL 文本、代号。等宽是"机器事实"的视觉信号。
- 字阶：12/13/14（密集表格 13）/16/20/24。行高紧凑（表格 1.4）。

### 3.3 间距/圆角/阴影
- 间距 4 基数（4/8/12/16/24/32）。圆角 6（卡片/输入）、4（徽章）。阴影克制（暗色用边框 + 微弱内发光区分层级，少用投影）。

### 3.4 雪花 id 呈现纪律
- id 在协议里**恒为字符串**（`>2^53`，前端**绝不**当 number 解析，否则丢精度）。等宽显示，默认截断中段（`7300…0123`）+ 悬浮全展 + 一键复制。

## 四、核心组件库（统一，所有页面复用）

| 组件 | 用途 | 关键约定 |
|---|---|---|
| **AppShell** | 全局骨架 | 顶栏（品牌 + 全局应急区 + 主题切换 + daemon 健康灯）+ 左侧导航 + 内容区 + 右侧抽屉位 |
| **GlobalEmergencyBar** | 应急区（顶栏常驻） | **Freeze 切换** + 当前全局模式徽章；freeze 生效时整页顶部红色脉冲横幅 |
| **DataTable** | 密集数据表 | 列排序、列筛选、**强制分页**（页号/页大小，缺省 20/钳 200，对齐后端）、空态、加载骨架、行级操作菜单；雪花 id 列等宽截断 |
| **ResourceCodeBadge** | 资源代号 | 等宽 + adapter/transport 小图标；永不显真实地址 |
| **CapabilityBadge** | 能力动词徽章 | 固定配色（observe…destroy 色温递增）；只读 |
| **DecisionBadge** | allow/deny/escalate | 固定语义色 + 图标；deny 可展开 stage+reason |
| **StageChip** | deny 阶段 | 后端 `Stage` 闭集 10 值 `auth/classify/rbac/constraint/condition/tier/transport/exec/audit/discover` 固定色 + 顺序语义；只渲染闭集、**不编造阶段**（含不存在的 `connect`） |
| **ModeSelector** | 模式切换 | normal/observe/maintain/freeze 单选 + TTL 输入；freeze 高危确认 |
| **CredentialMetaCard** | 凭证元数据 | kind/可信域/有效期/吊销态;**绝不显 secret_hash/明文** |
| **FormDrawer / FormModal** | 创建/编辑写操作 | RHF+Zod 校验、提交前**摘要预览**、危险操作二次确认、409 冲突提示、成功提示 policy_rev 前进 |
| **ConfirmDialog** | 危险确认 | 破坏性/扩权/freeze/shutdown/import 覆盖；需输入确认或显式勾选 |
| **DenyResponseView** | 结构化拒绝展示 | decision/denied/reason/your_grants/request_hint/operator_note 逐字段；operator_note 原样、不加工 |
| **AuditEventRow** | 审计事件行 | 全信封字段；两阶段（intent/outcome）配对折叠；等宽 id/摘要 |
| **VerifyItemRow** | 红队项 | PASS（绿）/FAIL（红 + gap_note 指出缺口防线） |
| **EmptyState / ErrorState / LoadingSkeleton** | 三态 | fail-closed：错误态不显伪数据；加载用骨架 |
| **TtlBadge** | TTL 倒计时 | 临时授权/模式 TTL，临近过期转琥珀 |
| **JsonViewer / SqlText** | 机器事实展示 | 等宽、语法高亮、只读、可复制；SQL 仅展示不可编辑 |

## 七、统一交互模式（所有页面遵守）

- **列表页统一骨架**：标题 + 主操作（右上）+ 筛选条 + DataTable（强制分页）+ 行操作 + 右侧 FormDrawer 创建/编辑。
- **写操作统一流程**：表单（RHF+Zod）→ 提交前摘要预览 →（危险则）确认 → 乐观/失效刷新 → 成功提示（policy_rev 前进 + 可跳 audit）/ 失败红色错误（不改视图）/ 409 冲突（提示刷新）。
- **加载/错误/空**：骨架 / ErrorState（fail-closed，不显伪数据）/ EmptyState（带主操作引导）。
- **危险动作清单**（一律 ConfirmDialog）：授权扩权、destroy 单格授予、freeze、吊销凭证、shutdown、import 覆盖、删除（逻辑删除）。
- **键盘可达**：表格行焦点、`/` 聚焦筛选、`g a` 跳审计等（渐进增强，不阻塞 v1）。
- **可访问性**：语义 HTML、对比度达 AA、状态不仅靠色（带图标/文字）。

## 八、与后端契约对齐的硬约束

- 分页参数 `page_no/page_size`（缺省 20、钳 200），集合查询一律分页（对齐 `DB_PAGINATION_MANDATORY`）。
- 雪花 id 全程字符串、绝不丢精度。
- 写端点乐观锁：携带期望 version，409→提示刷新重试。
- deny 渲染逐字段、operator_note 原样、不暴露 Scope 外资源存在性。
- StageChip 只渲染后端 `Stage` 闭集 10 值（`auth/classify/rbac/constraint/condition/tier/transport/exec/audit/discover`，对齐 postern-core `error/stage.rs`，**该 enum 非 `#[non_exhaustive]`**）；前端**绝不编造**阶段（如 `connect`），各页阶段措辞均以本基座为单一事实源。
- 凭证/资源：永不显 secret_hash/明文/真实地址。
- 一切错误 fail-closed 呈现。
