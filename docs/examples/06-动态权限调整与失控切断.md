# 场景 6 · 动态权限调整与失控切断

> 运行期的"方向盘与刹车"：临时提权带 TTL 自动回收、模式一键切换（normal/observe/maintain/freeze）、发现某 Agent 失控时即时切断（按身份吊销凭证 / 按操作冻结模式），全部经控制面事务写入热生效，不断已建隧道、不重启 daemon。本场景把"安全不依赖实时审批，但安全可被实时收紧"的运行期治理钉死预期。

> 资源代号、Agent 名、角色名沿用 [01-多资源多Agent授权示例](01-多资源多Agent授权示例.md)（`svc-*`/`db-*`/`docker-A`/`redis-main`/`agent1..3`/`observer`/`operator`/`maintainer`）。一切真实地址、账号、令牌均以 `vault://` 引用或代号出现，绝不写明文（公理四、11.2 不变量）。

---

## 1. 用户需求

我已经把两机多资源、三个 Agent 的授权矩阵建好并稳定运行（场景 01/03/04）。现在我要的是**运行期的动态治理能力**，覆盖三类实际诉求：

1. **临时提权、到点自动收回**：agent2 临时需要对 `redis-main` 做一次平时不授予的 `destroy`（清一个坏掉的 key 空间），我想给它**开一个口子但带 30 分钟 TTL**——到点自动回收，不用我记得手动收，更不能因为我忘了而长期留权。
2. **模式一键切换**：上线一个高风险变更前，我想把全局或某资源切到 **observe（只读观察）** 看动静；做维护时切到 **maintain**；出问题时一键 **freeze（全局冻结，谁都别动）**；处理完切回 **normal**。这些都得**热生效**，不能重启网关、不能断掉正在跑的只读监控。
3. **发现失控立即切断**：监控告警 agent3 在反复尝试越权、像是被注入了——我要能**即时切断它**。两种粒度：
   - **按身份切断**：直接**吊销 agent3 的网关凭证**，让它后续所有请求在认证阶段就被拒，而**不影响 agent1/agent2**。
   - **按操作切断**：把相关资源或全局**冻结**，让一切写/管/毁动作（甚至一切动作）被拒，但保留我自己排查用的只读视图（视粒度）。
   - 而且切断必须对**正在执行的长操作**也生效——不是等它优雅跑完，是**强制砍断**在飞的危险操作。

痛点：传统做法要么改配置重启（断服务、丢隧道）、要么靠人盯着手动收权（必然遗漏）、要么切断只对"下一个请求"生效而对"正在跑的那个危险查询"无能为力。本场景要解决的就是**热生效的动态策略 + 到期自动回收 + 两粒度即时切断 + 在飞强制中断**，且全程 fail-closed、全程进审计。

覆盖的系统核心功能：动态策略热生效（对状态存储的事务写入 + 快照原子替换）、临时授权默认带 TTL 到期自动回收（求值时刻墙钟即刻失效 + sweeper 留痕）、模式（固定集合 normal/observe/maintain/freeze，可带 TTL）、失控切断两粒度（吊销凭证 / 冻结模式）、在飞请求中断语义、策略变更进审计。

## 2. 预期实现路径

所有动作都是**控制面动作**（人经 CLI/SPA 发起，写入策略库），数据面 Agent 既不可发起也不可见。控制面写端点的统一不变量是**一次事务 + 快照重建 + 审计事件三联动**（详细设计 6.5、模块 06 §3.4），写入即热生效。

### 2.1 临时提权与 TTL 自动回收

- **写入**：`postern elevate <principal> --cap <res:verb> --ttl <dur>` → `POST /v1/grants/temp`（6.5）。在 `temp_grants` 表（5.2，字段 `principal_id, resource_id, capability, granted_at, expires_at, ended_at, end_reason`）插入一行，`expires_at = now + ttl`（绝对墙钟时刻，非倒计时）。事务 COMMIT 后在同一写锁内重建 `PolicySnapshot`（6.2），临时授权格即刻进入"当前生效授权"集合：`bindings×role_capabilities ∪ temp_grants(expires_at >= now 且 ended_at 为空)`（详细设计 line 486）。
- **求值生效**：下一次求值在 RBAC 步骤[3] 即可命中该临时格；条件层步骤[5] 的 `ttl`（临时授权 TTL 有效性）谓词（技术设计求值管线步骤[5] 内置谓词集，5.2 关键判定原则"条件层是可扩展的谓词集合"）在**求值时刻按墙钟二次校验** `expires_at >= now`（详细设计 6.2 显式不变量）。
- **到期回收（双保险）**：
  - **正确性不依赖 sweeper 时序**：到 `expires_at` 之后的任一求值，墙钟二次校验立即判 `expires_at < now` → 该临时格对求值**即刻不可见**（行为契约 `TEMP_GRANT_EXPIRY_EFFECTIVE`，详细设计 line 1089）。即便此刻 sweeper 还没跑、快照还没因任何写入而重建，提权也已失效。
  - **sweeper 留痕**：后台 sweeper（模块 06 §3.6）事务性把 `expires_at < now` 的临时授权回收，写 `ended_at` 与 `end_reason='expired'`（与人工吊销的 `end_reason='revoked'` 语义分明，详细设计 line 508–511），actor=`system`，写 `policy_change` 审计；回收后同一写锁内重建快照（口子自动愈合的可见记录）。

### 2.2 模式切换（normal/observe/maintain/freeze）

- **写入**：`postern mode set <mode> [--resource <code>] [--ttl <dur>]` → `PUT /v1/mode`（6.5）；`postern freeze` = `mode set freeze`（全局）的便捷别名（模块 07 §3）。写 `mode_state` 表（5.2，`scope_resource_id` NULL=全局，`mode CHECK(normal/observe/maintain/freeze)`，`expires_at`）。切换是同辖区行的**事务内 UPSERT**（详细设计 line 527–531），`normal` 无需显式存行（辖区无生效行即 normal）。可带 `--ttl`（如冻结 1 小时后自动回落），由 sweeper 到期回收 `mode_state.expires_at` 并写 `mode_change`。
- **固定集合、内置语义**：模式集合**固定**为四个，是系统内置概念，不可由策略新增（CONS-18，详细设计 line 533–536）。各模式对动词的覆盖规则（`observe` 只放行只读动词、`maintain` 限维护类、`freeze` 拒绝一切）固化于 **core 内常量表**，求值步骤[5] 的 `mode` 谓词引用该常量表判定，**不从策略库读"模式定义"**——模式语义不漂移、不可被数据扩展。
- **求值生效**：求值时"当前生效授权"先按 RBAC/temp_grants 算出，再**与 `mode_state` 求交**（详细设计 line 486）。`mode` 谓词在步骤[5] 按墙钟二次校验 `mode_state.expires_at` 并取当前辖区生效模式收窄动词集。
- **fail-closed 兜底**：万一同辖区出现多行生效模式（全局行 NULL 不参与唯一索引的历史隐患，详细设计 line 480–483），快照构建取**最严格者**（`freeze > maintain > observe > normal`）并写告警审计，**绝不取最宽松者**。`mode_state` 是限制性表，**禁用 enable_flag**（`CHECK(enable_flag=1)`，详细设计 line 491–494）——解除冻结只能经显式 `mode set normal`，各写专门的 `mode_change` 审计，绝不悄无声息地翻 flag。

### 2.3 失控切断两粒度

- **按身份吊销凭证**：`postern credential revoke <id>` → `POST /v1/credentials/{id}/revoke`（6.5）。`credentials` 表写 `revoked_at`（领域语义：吊销≠删除，详细设计 line 462），写 `credential_event` 审计。求值时该凭证在**认证步骤[1]** 即被拒（`revoked_at` 在求值时刻按墙钟二次校验，行为契约 `EXPIRED_OR_REVOKED_CREDENTIAL_DENIED`）——等同未认证，根本走不到 RBAC。**精确到该凭证**：同一 Principal 的其他凭证、其他 Principal 的凭证均不受影响（per-credential 终态字段）。
- **按操作冻结**：`postern freeze` 或 `mode set freeze --resource <code>`（见 2.2）。不动凭证、不动授权矩阵，只把模式收窄——求值步骤[5] 的 `mode` 谓词把动词集收到空（freeze）或只读（observe）。

### 2.4 在飞请求的强制中断（核心难点）

freeze/吊销经快照原子替换对**尚未开始执行的后续请求**即时生效（下一次 `evaluate` 读到新快照即拒），但对**已进入执行阶段的在飞操作**，快照替换**不保证中止**底层正在跑的查询/隧道——优雅排空只是"不再接新"，不等于"砍断在飞"（详细设计 6.2 line 761–766）。

- **真正的中断手段**：freeze/吊销时，**连接管理层**对相关 `(resource[, principal])` 的在用连接**强制 abort/cancel**——取消底层查询、关闭隧道，而非仅优雅排空（详细设计 line 765，模块 06 §3.5 line 89）。
- **可打断性**：对长操作设置可被控制面信号打断的检查点或超时，使"已开始的危险操作"存在可控中止路径（详细设计 line 766）。
- **审计**：连接强制中断写 `connection_event`（resource、tier 名、transport 种类，不含真实地址/凭据，模块 06 §3.5 line 91）。

### 2.5 daemon 重启恢复

TTL 以**绝对墙钟 `expires_at`** 持久化在 `policy.db`（非内存倒计时）。daemon 重启时启动序列 boot（模块 06 §3.1）开库 → 重建首份快照 → ……；重建后求值仍以求值时刻 `expires_at >= now` 判定——**剩余 TTL 继续计时，不重置、不延长**。冻结/吊销同理：`mode_state`/`credentials.revoked_at` 持久化，重启后继续生效。

## 3. 页面与操作

### 3.1 CLI（`postern ...`，人/脚本用）— 主入口

| 诉求 | 命令 | 控制面端点 |
|---|---|---|
| 临时提权带 TTL | `postern elevate agent2 --cap redis-main:destroy --ttl 30m` | `POST /v1/grants/temp` |
| 主动撤销临时授权 | `postern revoke-grant <grant_id>` | `DELETE /v1/grants/temp/{id}` |
| 切观察模式（全局） | `postern mode set observe` | `PUT /v1/mode` |
| 切观察模式（单资源） | `postern mode set observe --resource db-main` | `PUT /v1/mode` |
| 切维护模式（带 TTL） | `postern mode set maintain --resource docker-A --ttl 2h` | `PUT /v1/mode` |
| 一键全局冻结 | `postern freeze`（=`mode set freeze` 全局） | `PUT /v1/mode` |
| 解冻/回落 | `postern mode set normal [--resource <code>]` | `PUT /v1/mode` |
| 按身份切断（吊销凭证） | `postern credential revoke <credential_id>` | `POST /v1/credentials/{id}/revoke` |
| 查某 Agent 当前授权（确认切断效果） | `postern grants agent3` | `GET /v1/grants/{principal}` |
| 查切断/调权的审计留痕 | `postern audit --principal agent3 --kind policy_change` | `GET /v1/audit?...` |

写命令携带先前读取的期望 `version`（乐观锁，模块 07 §3、6.5）；冲突返回 `409 Conflict` 原样呈现，不静默重试。

### 3.2 管理前端 SPA（控制面 API 的视图，人用）

SPA 是 6.5 控制面端点的图形视图，本场景相关页面：

- **运行态总览 / 模式面板**：显示当前全局模式与各资源辖区模式（来自 `GET /v1/mode` 同源状态）；提供 normal/observe/maintain/freeze 的切换控件与 TTL 选择；醒目的"**一键冻结（Freeze）**"急停按钮（对应 `PUT /v1/mode` freeze）。切换后页面回显新模式与生效审计条目（`mode_change`）。
- **临时授权管理页**：列出当前生效的临时授权（`principal/资源代号/动词/granted_at/expires_at` 倒计时显示），提供"提权"表单（`POST /v1/grants/temp`，必填 `--ttl`）与"立即撤销"动作（`DELETE /v1/grants/temp/{id}`）。瞬时态不导出（6.6），但运行期视图可见。
- **主体与凭证页**：列出每个 Principal 的凭证（`credential_id/kind/expires_at/revoked_at`）；提供"**吊销**"动作（`POST /v1/credentials/{id}/revoke`），吊销后该行标注 `revoked_at` 与 `credential_event`，其余凭证不受影响。
- **授权视图页**：`GET /v1/grants/{principal}`——切断后刷新该 Agent 视图以确认其授权世界已收窄（与 Agent 自己看到的 `your_grants` 同源）。
- **审计页**：按 `kind=policy_change/mode_change/credential_event/connection_event` 过滤，确认每次调权/切断都有"是谁、何时、改了哪行、旧值→新值"的可见痕迹。

所有 SPA 写操作经同一控制面端点，享受同一事务+快照+审计三联动与乐观锁；SPA 自身无安全逻辑（8.12 外壳层）。

### 3.3 数据面（Agent 用，MCP 工具 / HTTP 外壳）

**对调权/切断动作本身：不适用**——动态权限调整与失控切断是**控制面**专属，数据面 Agent **无任何发起入口**（`postern_*` 工具集是固定动词工具 `postern_query/observe/mutate/execute/manage/destroy/grants/surface`，模块 06 §3.3，不含任何提权/吊销/模式工具；平面隔离使数据面无 `PolicyRepo`，模块 06 line 254）。

数据面在本场景的角色是**被切断方 / 被收窄方**，可观察到切断的效果：

- Agent 经 `postern_query/mutate/manage/...` 发起的后续请求，会因 TTL 到期/freeze/吊销而被拒，得到结构化 `DenyResponse`（见 4.1/4.2）。
- Agent 经 `postern_grants` / `postern_surface` 查到的自身授权世界会随调权热更新（提权后多出临时格、回收/冻结后收窄）——但 `postern_surface` 只是授权快照投影，不触达底层资源（CONS-20，模块 06 §3.3）。

## 4. 使用方式与预期结果

### 4.1 正常操作

#### A. 临时提权带 TTL → 热生效 → 到期自动回收

1. **操作**：运维执行 `postern elevate agent2 --cap redis-main:destroy --ttl 30m`。
   **预期结果**：返回 `201`，体含新临时授权 `id`（雪花字符串）、`principal=agent2`、`resource=redis-main`、`capability=destroy`、`expires_at=now+30m`（绝对墙钟）。`temp_grants` 插入一行（`end_reason` 为空），事务 COMMIT 后快照原子重建。写 `policy_change` 审计（actor=操作者、实体表名+行 id+写后 version、变更摘要）。**热生效**：无需重启、不断任何隧道。
2. **操作**：agent2 经 `postern_destroy(resource="redis-main", request=<清坏 key 空间>)` 发起一次（TTL 窗口内）。
   **预期结果**：求值步骤[3] 命中 `(redis-main, destroy)` 临时格；步骤[5] `ttl` 谓词按墙钟校验 `expires_at >= now` 通过；步骤[6] `Allow{tier=admin}`（redis admin ACL 承载 destroy，01 §1.2）；执行并脱敏返回。`request` 审计 `decision=allow, capability=destroy, tier=admin`。
3. **操作**：等过 30 分钟，agent2 再次发起同样的 `postern_destroy`。
   **预期结果**：求值步骤[5] `ttl` 谓词墙钟校验 `expires_at < now` → 该临时格**即刻不可见** → 步骤[3] 已无 destroy 格 → `deny`。**不依赖 sweeper 是否已跑**（行为契约 `TEMP_GRANT_EXPIRY_EFFECTIVE`）。返回结构化 `DenyResponse`（`reason` 引用策略事实，`your_grants` 已不含该临时格）。
4. **操作**：（无人工干预，等 sweeper 周期）。
   **预期结果**：sweeper 事务性把该过期行回收，写 `ended_at`、`end_reason='expired'`，actor=`system`，写 `policy_change` 审计；回收后快照重建。审计可区分"自然过期（expired）"与"人工吊销（revoked）"。
5. **（变体）主动提前撤销**：在 TTL 未到时执行 `postern revoke-grant <id>`。
   **预期结果**：`DELETE /v1/grants/temp/{id}` 成功，`temp_grants` 该行 `end_reason='revoked'`（人工，区别于 expired），快照重建，agent2 下一请求即无此格 → deny。

#### B. 模式切换 observe / maintain → 热生效

6. **操作**：上线高风险变更前，运维执行 `postern mode set observe`（全局）。
   **预期结果**：`mode_state` 全局行 UPSERT 为 `observe`，快照重建，写 `mode_change`（旧 `normal` → 新 `observe`、scope=全局、ttl=空）。**热生效**：正在跑的只读监控（agent1 的 observe/query）**不受影响、隧道不断**。
7. **操作**：agent3 在 observe 模式下经 `postern_mutate(resource="svc-order", ...)` 试图下单（其 operator 角色平时可做）。
   **预期结果**：求值步骤[3] RBAC 命中（agent3 有 svc-order:mutate 格）但步骤[5] `mode` 谓词判定全局 observe **只放行只读动词** → mutate 被收窄掉 → `deny`，`stage=condition`，`reason` 引用"当前模式 observe 不放行 mutate"。agent1 的只读请求照常 allow。
8. **操作**：维护 `docker-A` 时执行 `postern mode set maintain --resource docker-A --ttl 2h`。
   **预期结果**：`docker-A` 辖区行 UPSERT 为 `maintain`，`expires_at=now+2h`，快照重建，写 `mode_change`。该辖区放行维护类动词；2 小时后 sweeper 回收 `mode_state.expires_at`，辖区回落到上层默认（全局模式或 normal），写 `mode_change`（到期回退）。
9. **操作**：处理完，执行 `postern mode set normal`（全局）。
   **预期结果**：全局行 UPSERT 回 `normal`（或显式删除该辖区覆盖回落），快照重建，写 `mode_change`。解除限制经**显式 mode set**留下可见痕迹（绝非翻 enable_flag）。

#### C. 一键拉闸（freeze）→ 切断在飞

10. **操作**：发现异常，运维点 SPA 急停或执行 `postern freeze`。
    **预期结果**：`mode_state` 全局行 UPSERT 为 `freeze`，快照重建，写 `mode_change`。对**尚未开始执行的请求**即时生效——下一次任何 Agent 任何动词（含只读）的 `evaluate` 读到新快照，步骤[5] `mode` 谓词判 freeze**拒绝一切** → `deny`。
11. **操作**：此刻恰有一条 agent2 的长 `destroy` 在飞（已进入执行阶段）。
    **预期结果**：快照替换不"砍"它，但**连接管理层对相关 `(resource[, principal])` 在用连接强制 abort/cancel**——取消底层查询、关闭隧道（非优雅等待其跑完）。写 `connection_event`（resource、tier 名、transport 种类，不含真实地址/凭据）。在飞危险操作被**强制中断**，不是优雅排空。
12. **操作**：处理完毕，执行 `postern mode set normal` 解冻。
    **预期结果**：全局行 UPSERT 回 `normal`，快照重建，写 `mode_change`；后续请求恢复正常求值。

#### D. 失控某 Agent → 按身份吊销凭证

13. **操作**：告警 agent3 疑似被注入，运维先 `postern grants agent3` 确认其授权面，再 `postern credential revoke <agent3 凭证 id>`。
    **预期结果**：`credentials` 该行写 `revoked_at`，写 `credential_event`，快照重建。**精确到该凭证**：agent1/agent2 的凭证、agent3 的其他凭证（若有）均不受影响。
14. **操作**：agent3 用被吊销凭证经任意 `postern_*` 工具发起后续请求。
    **预期结果**：求值**步骤[1] 认证** `revoked_at` 墙钟二次校验 → 凭证视为不可用 → `deny`（等同未认证，根本不进 RBAC，行为契约 `EXPIRED_OR_REVOKED_CREDENTIAL_DENIED`）。`stage=auth`。返回 `DenyResponse` 经同一 Sanitizer 出口脱敏。
15. **操作**：agent1 同时正常发起只读请求。
    **预期结果**：agent1 凭证未吊销 → 认证通过 → 正常 allow。**吊销只切 agent3，不波及他人**。

#### E. daemon 重启后 TTL 继续计时

16. **操作**：A 步给 agent2 提权 30m 后第 10 分钟，daemon 因运维原因重启。
    **预期结果**：boot 开库 → 从 `policy.db` 重建首份快照 → 临时授权行（`expires_at` 为提权时刻+30m 的绝对墙钟）随快照恢复。**剩余约 20 分钟继续计时，不重置、不延长**。重启后第 25 分钟（原 TTL 内）agent2 请求仍 allow；第 31 分钟（超原 TTL）即 deny。冻结/吊销同理：`mode_state`/`revoked_at` 持久化，重启后继续生效。

### 4.2 异常场景与预期结果（一律 fail-closed）

| # | 触发条件 | 系统行为 | 预期结果 |
|---|---|---|---|
| 1 | **TTL 到期但 sweeper 尚未运行**：agent2 在 `expires_at` 之后、sweeper 未回收前发起 destroy | 求值步骤[5] `ttl` 谓词按**求值时刻墙钟**二次校验 `expires_at < now`，临时格即刻不可见（详细设计 6.2、行为契约 `TEMP_GRANT_EXPIRY_EFFECTIVE`） | **即刻 deny**，不依赖 sweeper 时序；`DenyResponse.your_grants` 已不含该格；sweeper 稍后回收只是留痕（`end_reason=expired`），不影响此判定 |
| 2 | **freeze 下一切被拒**：全局 freeze 期间 agent1 发起只读 `postern_query` | 步骤[5] `mode` 谓词判 freeze 拒绝一切动词（含只读，内置常量表语义，CONS-18） | **deny**，`stage=condition`，`reason` 引用"当前模式 freeze"；不放行任何动词，fail-closed |
| 3 | **吊销后该凭证后续请求认证即拒**：用已 `revoked_at` 的凭证发请求 | 认证步骤[1] 墙钟校验 `revoked_at` → 凭证不可用 → deny（不进 RBAC） | **deny**，`stage=auth`；响应经 Sanitizer 脱敏，不回显凭证值、不泄露其他 Principal/资源存在性（`DENY_RESPONSE_SCOPE_BOUNDED`） |
| 4 | **吊销不波及其他凭证**：吊销 agent3 凭证后 agent1/agent2 继续请求 | per-credential 终态字段，仅该 `credential_id` 写 `revoked_at` | agent1/agent2 **认证照常通过、请求照常处理**；切断精确到身份，不误伤 |
| 5 | **daemon 重启后 TTL 剩余继续计时**：重启后用原临时格请求 | boot 从持久 `policy.db` 重建快照，`expires_at` 是绝对墙钟时刻，求值仍 `expires_at >= now` 判定 | 原 TTL 内 **allow**，超原 TTL **deny**；TTL **不因重启重置或延长**（绝对时刻持久化） |
| 6 | **冻结/吊销时在飞长操作**：切断时有底层查询正在跑 | 连接管理层对相关 `(resource[, principal])` 在用连接**强制 abort/cancel**（非优雅排空，详细设计 6.2、模块 06 §3.5） | 在飞操作被**强制中断**（取消查询/关隧道），写 `connection_event`；不等其优雅跑完，"已开始的危险操作"被砍断 |
| 7 | **试图把 freeze 旁路为 normal（多行并存隐患）**：同辖区出现多行生效模式 | 快照构建 fail-closed 兜底取**最严格者**（`freeze>maintain>observe>normal`）并写告警审计（详细设计 line 480–483） | 取 **freeze**（最严格），绝不取最宽松的 normal；旁路不成立 |
| 8 | **试图悄悄停用一条冻结/细则（enable_flag 翻转）**：对 `mode_state` 写 `enable_flag=0` | 限制性表 `CHECK(enable_flag=1)`，基础仓储拒绝写非 1 值（详细设计 line 491–494） | **写入被拒**；解除冻结只能经显式 `mode set normal`，留 `mode_change` 审计；停用限制绝非无声 flag 翻转 |
| 9 | **on_timeout 恒 deny（与切断同源的 fail-closed）**：审批开启但超时未裁，或 daemon 重启时挂起审批 | `approval.on_timeout` 固定 `deny`，不可配 allow（导入校验与设置写入两处都拒；重启时一切挂起审批恒 deny，模块 06 line 256） | escalate/超时/重启一律 **deny**，绝不超时放行（公理二，`ESCALATE_FOLDS_TO_DENY`） |
| 10 | **乐观锁冲突**：两运维并发改同辖区模式 / 同临时授权 | 更新携带期望 `version`，`UPDATE ... WHERE version=?` 影响 0 行 → 版本冲突（5.1-②、6.5） | 返回 **`409 Conflict`** 并写 `policy_change` 审计（记录冲突）；不静默重试覆盖；调用方须重读最新 `version` |
| 11 | **数据面 Agent 试图自行提权/解冻/吊销**：Agent 在 MCP 工具里找提权入口 | 数据面工具集是固定动词工具（无提权/模式/吊销工具）；平面隔离使数据面无 `PolicyRepo`（模块 06 §3.3、line 254） | **无入口可达**——动态治理是控制面专属；Agent 经 `request_hint` 只能拿到一条**给人执行**的 `postern elevate ...` 命令（机械生成的事实，公理六），自身无法执行 |
| 12 | **控制面不可达（`control.sock` 缺席/无权/daemon 未运行）下发切断**：CLI 连不上 | CLI 报明确错误（daemon 不可达），无本地安全路径可回退（模块 07 §6.2） | 命令**失败且如实报错**；daemon 侧安全状态不因 CLI 缺席而改变；不静默"以为切断成功" |
| 13 | **传给 `mode set` 非法模式名 / `elevate` 缺 `--ttl`** | CLI clap 本地语法校验（模块 07 §2）+ `mode_state.mode CHECK`/端点校验（5.2） | **本地或端点拒绝**；模式集合固定四个不可新增（CONS-18）；提权必须带 TTL（无 TTL 不接受），fail-closed |
| 14 | **解冻/切回 normal 写失败（事务/审计任一失败）** | 写端点三联动（事务+快照+审计）任一失败即整体失败，不假定部分生效（模块 07 §6.2） | 返回错误，**状态保持冻结**（更严格侧），运维须重试；绝不"以为解冻了实则仍冻"或反之留下未审计的状态 |
| 15 | **吊销/冻结后被切断 Agent 反复重试** | 每次重试都走完整求值，每次都在 auth/condition 阶段 deny，每条 deny 进审计（步骤[10]） | 持续 **deny**；高频拒绝在 `GET /v1/denials/summary` 聚合为告警信号（6.9），为运维提供"是否需进一步处置"的事实 |

> 一致性声明：本场景所有动作均为控制面写端点（6.5），遵循"一次事务 + 快照重建 + 审计"三联动与端到端乐观锁；TTL/模式/吊销的到期与失效一律以**求值时刻墙钟二次校验**为正确性来源、sweeper 仅做可见性回收（6.2）；在飞中断以连接管理层强制 abort 为真正手段（6.2、模块 06 §3.5）。返回 Agent 的一切（含拒绝响应）经同一 Sanitizer 脱敏、匿名化（仅资源代号）、只含事实（公理四/六），不确定即拒绝（公理二）。与上层冲突时以技术设计七公理与详细设计第八部分为准。
